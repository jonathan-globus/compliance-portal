Welcome to the Globus Compliance Portal. This site provides a centralized location for compliance documentation, security practices, and policy resources related to Globus services and data management workflows.

This landing page provides general information accessible to all visitors.  
Users who sign in with the appropriate permissions may access additional protected compliance and security documents.

## Globus Security Overview

Globus employs a comprehensive, multi-layered security model designed to protect data, identities, and services across the platform. Core elements include strong authentication and authorization, secure data transfer mechanisms, auditing and monitoring, and continuous security review.

Learn more:  
[Globus Security Overview](https://docs.globus.org/guides/overviews/security/)

---

## Product Security Practices

Globus maintains a formal product security program that includes secure software development lifecycle practices, vulnerability scanning, penetration testing, dependency and supply chain security, incident response processes, and continuous monitoring.

Learn more:  
[Globus Product Security](https://docs.globus.org/guides/overviews/security/product-security/)

---

## High Assurance (HA) Services

Globus High Assurance (HA) services support regulated and sensitive data environments, including HIPAA-aligned workflows. HA services provide enhanced identity verification, stricter session controls, detailed audit capabilities, and default encryption, enforced at the service-level, that cannot be bypassed.

Learn more:  
[Globus High Assurance Services](https://docs.globus.org/guides/overviews/high-assurance/)

---

## Sensitive Data Handling

Globus supports secure workflows for sensitive and regulated data, including data subject to institutional policies or external requirements such as HIPAA, NIH, or other research data protections. High Assurance collections and related controls help institutions meet their compliance obligations while enabling researchers to work efficiently.

For more information, see the dedicated page on Sensitive Data Handling:  
[Sensitive Data Handling](./sensitive-data-handling)
